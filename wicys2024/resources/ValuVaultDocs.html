About

ValuVault is the premier password management solution for enterprises, designed to seamlessly integrate into your existing IT infrastructure. As a robust command line tool, ValuVault empowers IT professionals and administrators to securely store, manage, and access passwords with unparalleled efficiency and control. Built with enterprise-grade security in mind, ValuVault utilizes state-of-the-art encryption technology to ensure your sensitive data is protected against breaches. Its powerful CLI capabilities enable streamlined automation, scripting, and integration into DevOps workflows, making it the ideal choice for businesses seeking to enhance their security posture. Trust ValuVault to provide the reliability, scalability, and security your enterprise demands.

Installing

Prerequisites

Before you begin, ensure you have:

  • Linux-based server (Ubuntu 20.04 or later recommended)
  • Root or sudo access to the server

Step 1: Download the Installer Script

Download the Installer Script Download the ValuVault installer script using wget: 

wget https://example.com/valuvault-installer.sh -O valuvault-installer.sh

Step 2: Run the Installer Script

Make the installer script executable and run it:

sh chmod +x valuvault-installer.sh
sudo ./valuvault-installer.sh

Step 3: Access ValuVault

Once the installation is complete, open a terminal on your client and access with: 

nc <your host> <your port>

Commands

LIST

Lists all avaliable passwords.

> LIST
passwordName1
passwordName2
passwordName3

GET <password name>

Gets a password for a supplied password name. User will need to supply the master vault password. 

> GET passwordName1
Enter master vault password:
password123

MOTD <string>

Echoes a test MOTD message.

> MOTD Here is the current time: {now.time}
Preview of your new MOTD:
Here is the current time: 19:54:09.307283

QUIT

Safely exits the ValuVault session.

> QUIT
Goodbye!

Configuration

Below are the configuration options available for ValuVault. These options should be specified in the configuration file (e.g., `valuvault-config.yaml`).

General Settings

server:
    port: 8080
    logLevel: info
    motd: "Welcome to ValuVault! Today's date is {now.date} and the time is {now.time}."

database:
    type: postgres
    host: localhost
    port: 5432
    user: valuvault
    password: securepassword
    name: valuvaultdb

security:
    encryptionKey: your-encryption-key
    jwtSecret: your-jwt-secret

Configuration Options

  • server.port: The port number on which the ValuVault server will run. Default is 8080.
  • server.logLevel: The level of logging. Possible values are info, warn, error. Default is info.
  • server.motd: The Message of the Day format string. Use {now.date} and {now.time} as placeholders for the current date and time.
  • database.type: The type of database. Default is postgres.
  • database.host: The hostname of the database server. Default is localhost.
  • database.port: The port number of the database server. Default is 5432.
  • database.user: The username for accessing the database.
  • database.password: The password for accessing the database.
  • database.name: The name of the database.
  • security.encryptionKey: The key used for encrypting data.
  • security.jwtSecret: The secret key used for signing JWT tokens.
